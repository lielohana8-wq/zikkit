# מדריך ממשק טכנאי — פיצ'רים

## לקוח לא עונה (No Answer)

### מה קורה כשטכנאי לוחץ "לא עונה":
1. סטטוס משתנה ל-**No Answer**
2. מונה ניסיונות עולה (1, 2, 3...)
3. **SMS אוטומטי נשלח ללקוח**: "ניסינו להשיג אותך, אנא חזור אלינו"
4. לוג עם תאריך ושעה של כל ניסיון
5. בעל העסק רואה בדשבורד כמה ניסיונות נעשו

### מה הטכנאי רואה:
- כפתור **📵 לא עונה — שלח SMS אוטומטי**
- ליד הכפתור — מספר הניסיונות (1), (2), (3)...
- אפשר לסנן עבודות לפי סטטוס "No Answer"

### מה בעל העסק רואה:
- בדשבורד — עבודות עם "לא עונה" מופיעות ב-Needs Attention
- בדף Jobs — סינון לפי No Answer
- בפרטי עבודה — לוג כל הניסיונות עם תאריכים

## מעקב חלקים (Parts Tracking)

### תהליך מלא:
1. טכנאי בעבודה — מגלה שצריך חלק
2. לוחץ **📦 הזמן חלקים** → נפתח מודל:
   - שם חלק (מדחס, פילטר, שסתום...)
   - ספק
   - עלות
   - צפי הגעה (ETA)
   - הערות (מספר הזמנה...)
3. סטטוס משתנה ל-**Waiting Parts**
4. כשחלקים מגיעים → טכנאי לוחץ **📦 חלקים הגיעו**:
   - סטטוס משתנה ל-**Parts Arrived**
   - **SMS אוטומטי ללקוח**: "החלקים הגיעו, נתאם מועד"
5. טכנאי מתאם וחוזר לעבודה

### סטטוסים בזרימה:
```
In Progress → Waiting Parts → Parts Arrived → In Progress → Completed
```

### מה נשמר בעבודה:
- רשימת חלקים (שם, ספק, עלות, ETA, סטטוס)
- תאריך הזמנה
- תאריך הגעה
- עלות חלקים (נכנסת לחישוב הרווח בסגירה)

---

## סגירת עבודה (Close Job)

### מה הטכנאי ממלא:
- סכום שגבה מהלקוח (Revenue)
- עלות חומרים (Materials)
- הערות סגירה

### מה המערכת מחשבת:
- **רווח** = Revenue - Materials
- עמלת טכנאי (לפי אחוז שהוגדר)
- נכנס לדוחות ול-Payroll

## סטטוסים מלאים

| סטטוס | צבע | משמעות |
|--------|------|---------|
| Assigned | כחול | הוקצה לטכנאי |
| Scheduled | כחול | מתוזמן |
| In Progress | צהוב | טכנאי בעבודה |
| Waiting Parts | סגול | ממתין לחלקים |
| Parts Arrived | ירוק | חלקים הגיעו |
| No Answer | אדום | לקוח לא עונה |
| Callback | צהוב | צריך לחזור ללקוח |
| Completed | ירוק | עבודה הושלמה |
| Dispute | אדום | מחלוקת |
